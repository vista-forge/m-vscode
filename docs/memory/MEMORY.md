## Lessons (durable)
- [⭐ Highlighting seams — every failure here is SILENT](highlighting-seams.md) — an unmapped capture, a filtered-out asset, a stale grammar and a grey theme all look identical, so each gets a gate (capture coverage is gated BOTH statically and empirically). ⚠️ web-tree-sitter columns are **UTF-16 code units** (measured, guarded by a non-ASCII fixture) — do NOT reuse the LSP path's byte conversion. Captures NEST; VS Code tokens must not → paint widest-first, innermost wins. Node kinds moved 1019→1020 unnoticed upstream (`> 900` assertion): a number quoted in three docs is not three confirmations. R1 measured GREEN — 183 steps, 26 ERROR trees, worst 554/556 tokens, 2.4 ms. Node-kind pin history: 1019 → 1020 (`8a3c0b2`, `_sp_comment`) → **1170** (`0d41453`, IRIS `$I`/`ZW`) — re-measure on every `make sync-wasm`; upstream's own `> 900` assertion never notices.
- [⚠️ vsix packaging gotchas](vsix-packaging-gotchas.md) — vsce REFUSES `.vscodeignore` + `files` together (peers disagree: atlas uses one, compass the other); verify a package offline by unzipping its `extension/package.json`, not by eye. **P2:** an unbundled runtime dep is invisible to every other gate → `scripts/verify-bundle.mjs`; `vscode-languageclient/node.js` needs the `.js` (no `exports` map).
- [⭐ Engine-feature seams — shelling to `m`](engine-feature-seams.md) — the envelope has TWO channels (failing `m test`: full report on stdout, short error on stderr; a refusal: stderr only) → parse stdout first, and a failing envelope is still a REPORT. ⚠️ `m coverage <suite>` alone measures NOTHING (positional paths are suites **and sources**; `[dependencies] routines` stages but does not measure) — exit 0 + empty tracefile renders as 0%, so "exited 0" is never evidence a measurement happened. `vista exec`/`status` take `--transport`, not `--docker`. ⚠️ A cross-repo drift gate must read the neighbour's **committed HEAD** — a dirty sibling worktree red this gate AND would have had us vendor uncommitted bytes into a release (inverse of [[docs-validate-worktree-masking]]). A defect planted in an unreached branch proves nothing.
- [⚠️ LSP client seams](lsp-client-seams.md) — where editor and CI diagnostics can silently disagree. ⭐ **Columns differ in ORIGIN _and_ UNIT** (`m lint` = 1-based BYTES, LSP = UTF-16 code units) — `character + 1` is right only on ASCII, so an ASCII-only fixture set made the equivalence gate structurally unable to fail on a live encoding bug. Convert through the DOCUMENT TEXT, never compare the two columns directly, and keep the non-ASCII fixture guard. Config discovery + formatting-preset seams are CLOSED (per-file on both sides; `[fmt] rules` honored) and now GATED — `fixtures/format/.m-cli.toml` must pin `canonical` or the write-side gate compares two no-ops. A gate that cannot fail is not a gate. ⭐ **Severity is MANY-TO-ONE on the wire** — `m lsp` publishes both `style` and `info` as Information (3) (Hint is excluded from the Problems panel, so style→Hint hid findings `--check` gates on) → normalise `m lint`'s NAME to the LSP NUMBER and compare numbers; never invent an inverse for a lossy mapping. ⭐ **P3-feat Session B (2026-07-20):** `vscode-languageclient`'s built-in features (Hover/CompletionItem/DocumentSymbol/FoldingRange) need zero client code — they self-register from the server's advertised capabilities. ⚠️ `transport: TransportKind.stdio` is NOT a no-op — for an `Executable` server it makes the library append `--stdio` to argv, which crashed the real `m lsp` (no such flag) with an error the equivalence gate structurally could not see (it bypasses `vscode-languageclient` entirely); omit `transport`. Only a real VS Code + `@vscode/test-electron` smoke suite (`src/smoke/`, not part of `make check`) caught it — a code-review hunch about `restart()` reentrancy was real but NOT the cause, confirmed (not assumed) via the smoke suite's own output-channel capture.
